# Оптимизация плагина

## ✅ Реализованная оптимизация

### 1. Многоуровневое кэширование

**Runtime кэш** - данные хранятся в памяти PHP на время выполнения скрипта:
```php
private static $runtime_cache = [];
```

**Object Cache** - использует встроенный WordPress Object Cache (совместим с Redis/Memcached):
```php
wp_cache_get($key, $group);
wp_cache_set($key, $data, $group, $expiration);
```

**Что кэшируется:**
- Все связки категорий (1 час)
- Подсчёты товаров для фильтров (1 час)
- Доступные атрибуты для категорий (1 час)

### 2. Автоматическая очистка кэша

Кэш очищается при:
- Сохранении/обновлении товаров
- Изменении терминов (категории, атрибуты)
- Обновлении настроек плагина

### 3. Ленивая загрузка данных

**Связки загружаются только при необходимости:**
```php
private function load_bindings() {
    if (!empty($this->bindings)) {
        return; // Уже загружены
    }
    // Загрузка...
}
```

### 4. Оптимизация WP_Query

**Отключены ненужные операции:**
```php
'fields' => 'ids',                    // Только ID
'no_found_rows' => false,             // Считаем только количество
'update_post_meta_cache' => false,    // Не кэшируем мета
'update_post_term_cache' => false,    // Не кэшируем термины
```

## 🚀 Дополнительная оптимизация (рекомендации)

### 1. Установите объектный кэш

**Redis (рекомендуется):**
```bash
# Установите Redis
sudo apt-get install redis-server

# Установите плагин Redis Object Cache
wp plugin install redis-cache --activate
wp redis enable
```

**Memcached:**
```bash
sudo apt-get install memcached php-memcached
```

### 2. Включите постоянное кэширование страниц

Используйте один из плагинов:
- **WP Rocket** (платный, лучший)
- **W3 Total Cache** (бесплатный)
- **WP Super Cache** (бесплатный)

### 3. Настройте кэширование на уровне сервера

**Nginx FastCGI Cache:**
```nginx
fastcgi_cache_path /var/cache/nginx levels=1:2 keys_zone=WORDPRESS:100m inactive=60m;
fastcgi_cache_key "$scheme$request_method$host$request_uri";
```

**Varnish Cache** - для высоконагруженных сайтов

### 4. Оптимизация базы данных

**Добавьте индексы для таксономий:**
```sql
ALTER TABLE wp_term_relationships ADD INDEX idx_object_term (object_id, term_taxonomy_id);
ALTER TABLE wp_terms ADD INDEX idx_slug (slug);
```

### 5. CDN для статических ресурсов

Используйте Cloudflare или другой CDN для:
- CSS/JS файлов плагина
- Изображений товаров

## 📊 Мониторинг производительности

### Проверка использования кэша

Добавьте в `functions.php`:
```php
add_action('shutdown', function() {
    if (defined('WP_DEBUG') && WP_DEBUG) {
        $cache_stats = wp_cache_get_stats();
        error_log('Cache Stats: ' . print_r($cache_stats, true));
    }
});
```

### Query Monitor

Установите плагин Query Monitor для отслеживания:
- Количества запросов к БД
- Времени выполнения запросов
- Использования кэша

```bash
wp plugin install query-monitor --activate
```

## ⚙️ Настройки плагина для производительности

### Увеличение времени кэширования

Для сайтов с редкими обновлениями товаров:

```php
// В class-wsf-cache-manager.php измените:
const CACHE_EXPIRATION = 86400; // 24 часа вместо 1 часа
```

### Отключение подсчёта товаров

Если подсчёт не критичен (для очень больших каталогов):

В `class-wsf-filter-widget.php`:
```php
private function get_filter_product_count($attribute, $value, $current_category_slug = '') {
    return 0; // Отключает подсчёт
}
```

## 🔧 Оптимизация для конкретных случаев

### Большой каталог (10000+ товаров)

1. **Используйте AJAX подгрузку фильтров** вместо серверного рендеринга
2. **Ограничьте количество значений** в фильтрах (топ-20)
3. **Включите Redis/Memcached** обязательно

### Много атрибутов (20+ фильтров)

1. **Разбейте фильтры на группы** (базовые/расширенные)
2. **Используйте выпадающие списки** вместо чекбоксов для больших списков
3. **Lazy loading** для второстепенных фильтров

### Высокая нагрузка

1. **Полностраничное кэширование** через Varnish
2. **Статические HTML** для популярных категорий
3. **Отдельный сервер** для базы данных

## 📈 Ожидаемые результаты

### До оптимизации:
- Загрузка страницы: 2-3 секунды
- Запросов к БД: 50-100
- Время генерации фильтров: 500-1000мс

### После оптимизации (с Redis):
- Загрузка страницы: 0.5-1 секунда
- Запросов к БД: 10-20
- Время генерации фильтров: 50-100мс

### С полным кэшированием:
- Загрузка страницы: 100-300мс
- Запросов к БД: 0 (из кэша)
- Время генерации фильтров: 0 (из кэша)

## 💡 Советы

1. **Тестируйте на продакшене** - результаты на локальном сервере могут отличаться
2. **Мониторьте** - используйте New Relic или другие APM
3. **Постепенно** - внедряйте оптимизации поэтапно
4. **Измеряйте** - всегда замеряйте результаты до и после

Плагин уже оптимизирован для большинства случаев использования. Дополнительные меры нужны только для высоконагруженных сайтов.
