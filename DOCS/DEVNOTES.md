# Важные заметки для разработки

## При работе с фильтрами
1. Всегда проверяй активность фильтра в query параметрах И в категории
2. При поиске категорий сначала ищи точное совпадение, потом частичное
3. Не забывай `array_map('trim', $values)` для query параметров
4. Используй кэш для подсчёта товаров

## При работе с админкой
1. Все данные сохраняются по slug, не по ID
2. Ajax методы должны проверять nonce и права
3. Drag & Drop требует jQuery UI Sortable
4. Прилипающая кнопка контролируется через `position: fixed`
5. Связки можно сворачивать/разворачивать кликом на заголовок (состояние в `data-collapsed`)
6. Дублирование связок копирует категорию, родителя и все атрибуты

## При работе с виджетами
1. Universal виджет берёт атрибуты из `wsf_category_available_attributes`
2. Badges виджет берёт атрибуты из `wsf_category_badge_attributes`, с fallback на `wsf_category_available_attributes`
3. Оба виджета используют наследование от родительской категории
4. Сортировка: активные → с товарами → пустые
5. Пустые неактивные = `<span class="disabled">`, активные всегда = `<a>`
6. **Badges виджет — сворачивание меток:**
   - HTML структура: `.wsf-badge-items-wrapper` содержит `.wsf-badge-items` (метки) и `.wsf-badge-toggle` (кнопка)
   - CSS: класс `.collapsed` добавляет `max-height: 28px` (высота одной строки) для `.wsf-badge-items`
   - Wrapper — flex-контейнер, размещает метки слева и кнопку справа
   - Кнопка всегда видна (не скрывается overflow'ом контейнера меток)
   - JS функция `initBadgeToggle()` сравнивает `scrollHeight` с 28px
   - Если метки переполняются — показывается кнопка, если нет — кнопка скрыта
   - Кнопка имеет `data-action="expand|collapse"` для переключения состояния
   - При resize окна вызывается `initBadgeToggle()` для пересчета

## При отладке
1. Проверь кэш: `wp cache flush` или деактивация/активация плагина
2. Проверь консоль браузера на JS ошибки
3. Проверь `/wp-content/debug.log` (если включён WP_DEBUG)
4. Используй `var_dump($bindings)` в шаблонах админки

## Оптимизация: многоуровневое кэширование

1. **Runtime cache** — в памяти PHP (самый быстрый)
   ```php
   private static $runtime_cache = [];
   ```

2. **WordPress Object Cache** — совместим с Redis/Memcached
   ```php
   wp_cache_get($key, 'wsf_filters');
   wp_cache_set($key, $data, 'wsf_filters', 3600);
   ```

3. **Автоочистка** — при изменении товаров/терминов/настроек

## Оптимизированные запросы

```php
'fields' => 'ids',                    // Только ID
'no_found_rows' => false,             // Считаем только количество
'update_post_meta_cache' => false,    // Не кэшируем мета
'update_post_term_cache' => false,    // Не кэшируем термины
```
