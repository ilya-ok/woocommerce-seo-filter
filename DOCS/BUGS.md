# Исправленные проблемы WooCommerce SEO Filter

### Проблема 1: Дублирование атрибутов в функции wsf_build_category_options()
**Решение:** Функция объявлена один раз в settings.php, используется в binding-item.php

### Проблема 2: Атрибуты не сохраняются при обновлении страницы
**Решение:** Предзаполнение значений атрибутов из БД при загрузке страницы

### Проблема 3: Зависимость от родительской категории на странице Filter Bindings
**Решение:** Убрали зависимость, атрибуты берутся из настроек в Filter Attributes

### Проблема 4: Двойной клик для добавления атрибута
**Решение:** Select всегда видимый с placeholder "Добавить атрибут..."

### Проблема 5: Связки перестали работать
**Решение:** Добавлен вызов `$this->load_bindings()` в начало всех методов

### Проблема 6: Дублирование фильтра "Резиновое покрытие" три раза
**Решение:** Добавлен `array_unique()` в `get_available_attributes_for_category()`

### Проблема 7: Неправильное удаление фильтров в виджете активных фильтров
**Решение:** Показываем только атрибуты из текущей категории, не из иерархии

### Проблема 8: Неправильный переход при снятии фильтра
**Решение:** Поиск лучшего совпадения среди всех SEO-категорий, не только родителя

### Проблема 9: Фильтр в query не распознаётся как активный
**Решение:** Проверка активности в query параметрах, приоритет query над категорией

### Проблема 10: Не учитываются query параметры при добавлении фильтра
**Решение:** Объединение атрибутов из категории + query + новый выбранный

### Проблема 11: Виджет SEO Filter Badges не отображается на вложенных категориях
**Решение:** Добавлена логика наследования атрибутов от родительской категории + fallback на атрибуты для фильтров

### Проблема 12: Слишком много меток в виджете Badges занимают много места
**Решение:** Реализовано автоматическое сворачивание в одну строку с кнопками "Показать еще"/"Свернуть"

**Детали реализации:**
- **Файлы изменены:**
  - `includes/class-wsf-badges-widget.php` — кнопка toggle добавлена внутрь `.wsf-badge-items`
  - `assets/css/frontend.css` — добавлены стили для collapsed состояния и позиционирования кнопки
  - `assets/js/frontend.js` — добавлена логика определения overflow и управления кнопками

- **HTML структура:**
  ```html
  <div class="wsf-badge-group">
    <span class="wsf-badge-label">Атрибут:</span>
    <div class="wsf-badge-items-wrapper">
      <div class="wsf-badge-items collapsed">
        <!-- badges -->
      </div>
      <button class="wsf-badge-toggle" data-action="expand">Показать еще</button>
    </div>
  </div>
  ```

- **CSS ключевые классы:**
  - `.wsf-badge-items-wrapper` — flex-контейнер для меток и кнопки
  - `.wsf-badge-items.collapsed` — ограничивает высоту до 28px (одна строка)
  - `.wsf-badge-toggle` — кнопка всегда видна справа от контейнера меток
  - Wrapper размещает метки слева (flex: 1) и кнопку справа (flex-shrink: 0)

- **JavaScript логика:**
  - `initBadgeToggle()` — проверяет каждую группу меток на overflow
  - Сравнивает `scrollHeight` (полная высота) с 28px (видимая высота)
  - Показывает/скрывает кнопку в зависимости от overflow
  - Обработчик клика переключает класс `collapsed` и текст кнопки
  - Обработчик `resize` пересчитывает при изменении размера окна

### Проблема 13: Фильтры в сайдбаре нужно сделать sticky (прилипающими)
**Решение:** Реализован умный sticky sidebar в `wp-content/themes/sportshop/js/common.js`

**Детали реализации:**
- **Файл:** `wp-content/themes/sportshop/js/common.js` (строки 642-824)
- **Объект:** `StickySidebar` с методами init, recalculate, update, handleShortSidebar, handleTallSidebar

**Логика работы:**
1. **Короткий сайдбар (помещается в экран):**
   - Прилипает к верху экрана при скролле вниз
   - Останавливается внизу контейнера

2. **Высокий сайдбар (больше высоты экрана):**
   - При скролле вниз: прилипает к низу экрана, пока не достигнет конца
   - При скролле вверх: прилипает к верху экрана
   - Отслеживает направление скролла и плавно перемещается внутри контейнера

**Особенности:**
- Использует `requestAnimationFrame` для плавности
- Учитывает padding контейнера при расчётах
- Position: fixed при прилипании к краям экрана
- Position: absolute при достижении границ контейнера
- Корректировка -100px в нижней позиции для учёта padding контейнера

**Инициализация:** Автоматически при `$(document).ready()`
